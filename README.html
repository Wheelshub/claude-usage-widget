Architecture

How it works

A widget on the Edge is sandboxed HTML — it can't read your Claude logs or token directly. So a tiny local collector gathers the data and serves it over localhost; the widget just draws it.

📡

Collector

A small Node server polls your usage and exposes a normalized usage.json on localhost — and can serve the widget itself.

🎯

Widget

Responsive HTML/CSS/JS rendered in iCUE's Chromium engine. Fetches the JSON and animates the rings + countdowns.

🔌

Pluggable

Flip one config value to switch between fully-local estimates and Anthropic's exact official numbers.

Pick your data

Two ways to feed the rings

Both are built in. The widget badge always tells you which you're seeing — OFFICIAL or ESTIMATED.

No setup

ccusage · default

ESTIMATED

Reads your own Claude Code logs locally via the ccusage tool.

  • ✓ 100% local — no token handling, no ToS concern
  • ✓ Tokens, cost, 5h block + reset, burn rate, projection
  • ✕ % is estimated against a token budget
  • ✕ No official weekly reset time
Exact

oauth · flip a flag

OFFICIAL

Calls Anthropic's /api/oauth/usage — same numbers as /usage.

  • ✓ Exact 5h %, weekly %, per-model Opus/Sonnet
  • ✓ Official reset timestamps
  • ✕ Undocumented endpoint; sends a Claude Code User-Agent (mild gray area)
  • ✕ Reads your local OAuth token; poll ≥ 180s
// collector/config.json — switch any time, then restart
{ "provider": "ccusage"   // or "oauth" }
Get running

Quick start

Default provider is ccusage, so there's nothing to configure. Two steps to pixels on glass.

Start the collector

Leave it running (or auto-start it at login — see below).

powershell -ExecutionPolicy Bypass -File scripts\start-collector.ps1

Add it to your dashboard

Easiest path, zero packaging — add an iFrame widget in iCUE and point it at:

http://127.0.0.1:8787/

(Optional) Install as a real .icuewidget

Builds Claude-Usage.icuewidget, then import in iCUE via Widgets → “+”.

powershell -ExecutionPolicy Bypass -File scripts\pack.ps1

(Optional) Run automatically at login

Adds a hidden launcher to your Startup folder (no admin) so the collector starts at login and the widget always has data.

powershell -ExecutionPolicy Bypass -File scripts\install-startup.ps1
# remove with: install-startup.ps1 -Remove

Everything lives in collector/config.json.

When something's off

Troubleshooting

⚠
Widget says “Collector unreachable”
The collector isn't running or the port differs. Start it and check http://127.0.0.1:8787/health.
⚠
iFrame widget is blank
Confirm the URL is exactly http://127.0.0.1:8787/ — http, not https.
⚠
oauth shows 401
Token expired — run any Claude Code command to refresh it.
⚠
oauth shows 429
Raise refreshSeconds to ≥ 180.
⚠
Cost numbers look huge
ccusage reports API-equivalent cost, not what your subscription actually bills.
